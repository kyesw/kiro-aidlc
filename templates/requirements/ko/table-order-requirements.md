# 테이블오더 서비스 요구사항 정의서

> 🗑️ **안내**: `🗑️`가 표시된 줄은 작성 완료 후 삭제하세요. AI에게 전달하기 전에 가이드 텍스트를 제거하면 더 정확한 결과를 얻을 수 있습니다.

## 1. 프로젝트 개요

### 1.1 서비스 비전
디지털 주문 시스템을 통해 고객에게는 편리한 주문 경험을, 매장 운영자에게는 효율적인 운영 환경을 제공하는 테이블오더 플랫폼

### 1.2 핵심 가치 제안

**고객 관점**
- 대기 시간 없는 즉시 주문
- 직관적이고 사용하기 쉬운 인터페이스

**매장 운영자 관점**
- 인건비 절감 및 운영 효율성 증대
- 실시간 주문 관리 및 모니터링

**시장 차별화**
- 다양한 업종에 적용 가능한 유연한 구조
- 직관적인 관리자 도구

---

## 2. 서비스 구성

### 2.1 주요 구성 요소
- **고객용 인터페이스**: 테이블에서 고객이 사용하는 주문 화면 (브라우저에서 동작하는 웹UI 방식)
- **관리자용 인터페이스**: 매장 운영자가 사용하는 관리 화면
- **서버 시스템**: 주문 처리 및 데이터 관리
- **데이터 저장소**: 매장, 메뉴, 주문 정보 저장

## 부록: 용어 참고

| 용어 | 쉬운 설명 |
|------|-----------|
| API | 서버와 화면이 데이터를 주고받는 방식 |
| 실시간 | 새로고침 없이 화면이 자동으로 업데이트되는 것 |
| 세션 | 한 고객이 앉아서 나갈 때까지의 시간 단위 |
| 인증 | 누구인지 확인하는 절차 (로그인) |
| SSE | 서버에서 화면으로 자동으로 데이터를 보내주는 기술 |
